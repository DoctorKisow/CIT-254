LAB 3: SCANNING WITH NMAP
INSTRUCTIONS
The scanning lab is an immersive and hands-on exploration of offensive security techniques, employing the powerful tools installed on Kali Linux and the intentionally vulnerable Metasploitable2 Linux server. As students delve into the intricacies of network scanning, the primary focus lies on utilizing the versatile capabilities of nmap. This comprehensive exploration encompasses the scanning of hosts, identification of open TCP/UDP ports, discernment of operating systems, and meticulous examination of version and service attributes. The hands-on nature of this lab ensures that participants acquire theoretical knowledge and develop a practical understanding of the methodologies employed in cybersecurity assessments. Through this immersive experience, students are poised to enhance their proficiency in network reconnaissance and sharpen their abilities to secure and defend against potential threats in the dynamic realm of cybersecurity.

Part 2: Host Discovery

Perform a basic nmap host discovery scan without port scanning. Normally this would be used to scan an entire subnet (or larger), but for this lab, we're going to target specific IPs of interest.

  1. Target your Metasploitable2 VM's IP Address 192.168.1.x.

    ⬤ ⬤ ⬤ CODE

      $ sudo nmap -sn TARGET

     

    NOTE
      ●

    We are using sudo so that nmap can generate arbitrary network packets for this scan.
     

    TIP
      ●

    From the nmap documentation:
     

    "The default host discovery is done with -sn consists of an ICMP echo request, TCP SYN to port 443, TCP ACK to port 80, and an ICMP timestamp request by default. When executed by an unprivileged user, only SYN packets are sent (using a connect call) to ports 80 and 443 on the target. When a privileged user tries to scan targets on a local ethernet network, ARP requests are used unless --send-ip was specified." -- https://nmap.org/book/man-host-discovery.html

Part 4: UDP Port Scanning

  1. To detect active services, perform a nmap UCP port scan on the Metasploitable2 VM.

    ⬤ ⬤ ⬤ CODE

      $ sudo nmap -sU TARGET

     

    NOTE
      ●

    We are using sudo so that nmap can generate arbitrary network packets for this scan.
     

    TIP
      ●

    Unlike TCP, there is no generic way to see if a UDP port is open or not since UDP is connectionless. Thus, you will have much more accurate results (and be able to answer the lab question fully) if you also enable service and version scanning with your UDP scan. The CLI argument is provided below.
      ●

    If you accept the default options, this scan will take a long time. However, you can make nmap go faster by specifying a non-default Timing Template. Since the target is a VM on the same computer (even better than on the same local area network!), shorter timeouts should be perfectly safe.
