Part 1: Autonomous Systems (AS)

Autonomous Systems (AS) are networks of connected IP routing prefixes under the control of a single, clearly defined entity that presents a common routing policy to the Internet. Each Autonomous System is assigned a unique Autonomous System Number (ASN), a 16-bit identifier, or, in some cases, a 32-bit identifier, which facilitates the efficient exchange of routing information among interconnected networks on the Internet.

  1. The Community College of Allegheny County (CCAC) is no different; they use Autonomous System Numbers to route traffic between different services.

    1. Many online tools allow you to look up Autonomous System Numbers (ASN) information for a given organization name or domain name, such as:

      • MXToolbox - https://mxtoolbox.com/asn.aspx

         

        TIP
          ●

        Search for "162.51.74.111." Here, you can search by an IP Address used by the entity to identify the associated CIDR range for the ASN.

      • ANSLookup - https://asnlookup.com/

         

        TIP
          ●

        Search for "CCAC." Here, you can search by the registered entity name to identify the associated CIDR range for the ASN.

Classless Inter-Domain Routing (CIDR) is a method for allocating IP addresses and IP routing. CIDR notation is a compact representation of an IP address and its associated routing prefix. The notation is constructed from an IP address, a slash (/) character, and an integer.

  1. The CIDR entry 18.5.27.0/24 means that the upper 24 bits of the address (18.5.27) stay constant, and the lower 8 bits are variable and represent computers within the subnet. 18.5.27.0 is the first address, and 18.5.27.255 is the last address for 256 possible addresses in that range.

     

    NOTE
       

    If you are on campus while doing this lab, be careful! Your local system will see a private IP address for the sso.ccac.edu system of the form 10.x.x.x. For your ASN lookup, you want the public IP address of the ccac.edu system. Use an off-campus tool like MXToolbox to obtain that IP address.

Part 2: Shodan

Shodan is a search engine for finding specific devices and device types online. The most popular searches are for webcam, linksys, cisco, Netgear, SCADA, etc. It works by scanning the entire Internet and parsing the banners returned by various devices. Using that information, Shodan can tell you what web server (and version) is the most popular, how many anonymous FTP servers exist in a particular location, and what make and model the device may be.

  1. Create a Shodan Login (required to use most site features), then answer the deliverable questions for part two. To get the most out of Shodan, please refer to the references section at the end of part two.

Part 4: DNS

sublist3r is a DNS enumeration tool. It uses a combination of Internet search engines and (optionally) brute force guessing to provide a list of subdomains given a starting domain.

  1. Sign up for a free account at VirusTotal. After registering and logging in, copy the API key from your profile.

  1. Install sublist3r on your Kali Linux VM; and as of 2023 the code patch to fix integration with VirusTotal.

    ⬤ ⬤ ⬤ CODE

      # Install Sublist3r
      $ sudo apt install sublist3r

      # Patch Sublist3r: Follow the instructions on this repository.
      # https://github.com/aboul3la/Sublist3r/issues/194
      $ wget https://raw.githubusercontent.com/aboul3la/Sublist3r/3cb826c2f36f4972dfd286c704efc07de3a7f94c/sublist3r.py
      $ sudo mv sublist3r.py /usr/lib/python3/dist-packages/sublist3r.py

      # Configure your VirusTotal API key, which will be used to Sublist3r
      $ export VT_APIKEY="<your-api-key-copied-from-virustotal>"

  1. Using sublist3r, obtain an incomplete but lengthy list of subdomains for the domain name "ccac.edu."

    ⬤ ⬤ ⬤ CODE

      # Run the enumeration
      $ sublist3r --domain ccac.edu

Fierce is a domain name scanner to help locate non-contiguous IP space and hostnames against specified domains. In other words, given a domain name, it will find subdomains and locate nearby servers ("nearby" based on IP address) that may or may not share the same domain name. This tool does not perform exploitation and does not scan the whole Internet indiscriminately. It is meant to locate likely targets inside and outside a corporate network. Because it uses DNS primarily, you will often find misconfigured networks that leak internal address space.

  1. Use Fierce (in your Kali VM) to do a DNS scan:

    ⬤ ⬤ ⬤ CODE

      $ fierce --domain ccac.edu

Part 5: SSL Certificates

SSL certificates can be a valuable source of hostnames that may be of interest in a penetration test. Some certificates have a Subject Alternative Name (SAN), which is an extension to X.509 that allows various values to be associated with a security certificate using a subjectAltName field.

  1. You can inspect SSL certificates manually in your web browser or by using various online resources, such as:

    • https://andrewmohawk.com/SSLAssociated/index.php
    • https://www.ssllabs.com/ssltest/
